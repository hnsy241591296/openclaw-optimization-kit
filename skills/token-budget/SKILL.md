# Token 预算规则

## 回复构成目标
- 系统提示：由框架控制，无需干预
- 工具调用：每次最多调用3个工具
- 记忆注入：只注入搜索到的相关片段，不全量读取
- 回复正文：简单问题<100字，复杂任务<500字

## 工具调用节省
- 能一次调用完成的，不拆成多次
- 读文件前先判断是否真的需要
- 搜索前先判断是否已知答案

## 触发压缩的信号
- 用户反馈回复太长
- 任务已完成但还在继续输出
- 重复解释已经说过的内容

## 禁止行为
- 未经请求输出大段代码注释
- 回复中包含"总结一下""综上所述"等冗余段落
- 同一信息在回复中出现超过一次
